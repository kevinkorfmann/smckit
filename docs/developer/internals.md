# Internals and Porting Notes

This section holds the dense implementation material that does not belong in
the main user-facing method pages.

Use these pages when you are:

- porting an upstream method into smckit
- validating native vs upstream parity
- tracing a result back to source code or equations
- reviewing design decisions behind a specific implementation

If you are trying to run an analysis, start with the quickstarts and method
reference pages instead.

## Available internals pages

- [PSMC internals](internals-psmc.md)
- [SMC++ internals](internals-smcpp.md)
- [eSMC2 internals](internals-esmc2.md)
- [MSMC-IM internals](internals-msmc-im.md)

These pages intentionally keep the math, source maps, and porting details away
from the main reading flow so the primary docs can stay practical and readable.
